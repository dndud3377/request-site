# GUIDE — 전체 가이드 (중앙 모달 + 페이지별 애니메이션 데모)

## 개요
홈 화면 상단의 "전체 가이드" 버튼을 클릭하면, 화면 정중앙에 모달이 뜨고 실제 업무 흐름 순서대로 각 페이지를 소개한다. 서비스의 본질이 "요청서를 작성하고 결재를 진행하는" 전자결재형 워크플로우이므로, 가이드도 그 흐름을 그대로 따라간다.

**단계 순서 (내러티브 기반):**
1. 요청서 작성
2. 결재 진행
3. 이력 조회
4. VOC
5. 권한관리

각 단계는 정적 스크린샷이 아니라, 해당 페이지의 **핵심 동작 하나를 짧은 루프 애니메이션으로 자동 재생**하는 형태다 (예: 요청서 작성 단계에서는 실제로 입력 필드에 타이핑되고 제출 버튼이 눌리는 과정이 반복 재생됨). 사이드바 스포트라이트 같은 보조 장치는 사용하지 않고, 모달은 항상 화면 중앙에만 표시한다.

> ⚠️ **이 가이드는 기존 가이드 시스템과 별개로 새로 만드는 기능이다.**
> 현재 프로젝트에는 이미 (1) `/guide` 메뉴의 **기능별 가이드 지식베이스**(`pages/GuidePage.tsx`), (2) 우측 **슬라이드 패널**(`components/GuideSlidePanel.tsx`, framer-motion), (3) 단계별 **애니메이션 데모 컴포넌트**(`components/guideDemos/` — `Step1LineProcessDemo` 등 9개 + `GUIDE_DEMOS` 매핑)가 존재한다. 본 "전체 가이드"는 이들과 **목적·진입점·표시 방식이 다른 별도 기능**이며, 기존 `/guide` 메뉴를 대체하지 않는다. (단, 애니메이션 루프 구현 패턴/유틸은 기존 `guideDemos`를 참고·재사용할 수 있다 — "향후 확장" 참고)

프로토타입 참고: `docs/animated_demo_guide_mockup.html` (정적 HTML/CSS/JS, 단계별 애니메이션 루프 및 취소 로직 포함)

## 트리거
- **홈 화면 Hero 영역의 "전체 가이드" 버튼 클릭 (유일한 진입점)** — 이 버튼은 **신규로 추가해야 한다.** `pages/HomePage.tsx`의 Hero 섹션(의뢰서 작성 / 결재 현황 링크 버튼 영역)에 함께 배치한다. 기존 `/guide` 메뉴 링크(기능별 가이드 지식베이스)와는 별개 기능이다.
- 가이드는 **버튼을 눌렀을 때만 표시**된다. 자동 팝업/로그인 시 자동 시작은 없으며, 따라서 "다시 보지 않기" 및 노출 여부 영속성 저장도 두지 않는다.

## 데모 구현 방식 (결정됨)
"실제 페이지의 모습을 반영하면서" 영상 에셋 없이 구현하기 위해, **라이브 DOM 애니메이션으로 실제 페이지를 충실히 모사**하는 방식을 채택한다.
- 실제 페이지와 **동일한 CSS 변수/클래스(`styles/global.css`)를 공유**하여 디자인 시스템과 자동 동기화 → 페이지 디자인이 바뀌어도 영상 재녹화가 불필요
- mp4 영상 대비 선명도·i18n·테마 일관성이 유지되고, 일시정지는 `paused` prop으로 타이머만 멈추면 됨
- 실제 페이지 컴포넌트를 그대로 import하지 않는다 (API/데이터/라우터 의존성 때문). 대신 **실제 페이지의 마크업·스타일을 본뜬 전용 데모 컴포넌트**를 만든다.
- 기존 `components/guideDemos/`의 애니메이션 루프 패턴/유틸을 참고·재사용한다.

## 컴포넌트 구조 (React / TypeScript)
> 기존 가이드 데모와 구분하기 위해, 신규 파일은 접두사 `Tour*`를 사용한다 (기존 `guideDemos/`의 `Step*Demo`, `PermissionUserGroupDemo`와 이름 충돌 방지).

- Hero 버튼: `pages/HomePage.tsx` Hero 섹션에 "전체 가이드" 버튼 추가 → 클릭 시 `GuideTourModal` 오픈
- `components/GuideTourModal.tsx` — 오버레이 + 중앙 정렬 모달, step 상태(`current`)·일시정지 상태(`paused`) 관리
- `components/GuideTourStepPreview.tsx` — 현재 step에 해당하는 데모 컴포넌트를 렌더링하는 컨테이너 (high-level switch)
- 단계별 데모 컴포넌트 (각자 자체 애니메이션 루프 보유, `components/guideDemos/` 하위에 배치):
  - `TourRequestFormDemo` — 요청서 작성
  - `TourApprovalDemo` — 결재 진행
  - `TourHistoryDemo` — 이력 조회
  - `TourVocDemo` — VOC
  - `TourPermissionDemo` — 권한관리
- `useGuideTourSteps()` — 아래 데이터 모델 기반 step 메타데이터(제목/설명) 반환 (`hooks/` 또는 동일 컴포넌트 디렉토리)

## 데이터 모델
```ts
interface GuideTourStep {
  key: 'request' | 'approval' | 'history' | 'voc' | 'permission';
  title: string;
  description: string;
  DemoComponent: React.ComponentType<{ active: boolean; paused: boolean }>;
}
```
- `active` prop으로 현재 보여지고 있는 step인지 여부를 데모 컴포넌트에 전달 — 데모는 `active === true`일 때만 애니메이션 루프를 실행하고, `false`가 되는 즉시 정리(cleanup)한다.
- `paused` prop으로 일시정지 여부를 전달 — `paused === true`이면 애니메이션 루프를 현재 프레임에서 멈추고, `false`로 돌아오면 이어서 재생한다.

**step key ↔ 실제 라우트 매핑** (`components/Navbar.tsx`의 `NAV_LINKS` 기준):

| key | 라우트 | 페이지 컴포넌트 |
|---|---|---|
| `request` | `/request` | `pages/RequestPage/index.tsx` |
| `approval` | `/approval` | `pages/ApprovalPage.tsx` |
| `history` | `/history` | `pages/HistoryPage.tsx` |
| `voc` | `/voc` | `pages/VOCPage.tsx` |
| `permission` | `/permissions` (복수형 주의) | `pages/PermissionPage.tsx` |

> "시작하기"로 가이드를 종료할 때 특정 페이지로 이동시키려면 위 라우트를 사용한다. 특히 권한관리는 경로가 `/permissions`(복수)임에 유의.

## 애니메이션 루프 구현 가이드
- 각 데모 컴포넌트는 `useEffect(() => { ...; return cleanup; }, [active, paused])` 패턴으로 구현
- 루프 도중 step이 바뀌거나 모달이 닫히면 즉시 타이머/애니메이션을 정리해야 함 (겹쳐서 재생되는 현상 방지)
- 권장 패턴: `AbortController` 또는 단순 `let cancelled = false` 플래그를 클로저로 캡처 → `await sleep(ms)` 이후 매번 취소 여부 체크
- **일시정지**: `paused === true`인 동안에는 다음 시퀀스로 진행하지 않고 현재 상태를 유지한다. 재생 재개 시 처음부터 다시 시작할지(루프 재시작) 멈춘 지점부터 이어갈지는 데모 구현에 따르되, 단순 루프 데모는 "재개 시 현재 루프 이어서 진행"을 기본으로 한다.
- 반복되는 시퀀스(타이핑 효과, 배지 색상 전환, 버튼 펄스 등)는 공용 유틸로 분리해서 5개 데모 컴포넌트가 재사용:
  - `typeText(el, text, speed, signal)` — 타이핑 효과
  - `sleep(ms, signal)` — 취소 가능한 delay
  - `pulse(el)` — 버튼 클릭 펄스 애니메이션 클래스 토글
- CSS 트랜지션(opacity, transform, background-color)으로 시각적 전환을 처리하고, JS는 상태 변경 타이밍만 제어 — 실제 비디오 파일 대신 라이브 DOM 애니메이션으로 구현 (디자인 시스템과 항상 동기화되고, 별도 영상 에셋 제작/교체 비용이 없음)

## 단계별 데모 내용 (요약)
| 단계 | 핵심 동작 |
|---|---|
| 요청서 작성 | 유형 선택 → 제목/내용 타이핑 → 제출 버튼 클릭 → "제출 완료" 토스트 |
| 결재 진행 | 단계 스테퍼 진행 + 상태 배지가 대기중 → 검토중 → 승인완료로 전환 |
| 이력 조회 | 검색창 타이핑 → 일치 항목 강조, 비일치 항목 흐려짐 |
| VOC | 의견 작성 → 등록 → "답변대기" 배지가 잠시 후 "답변완료"로 전환 |
| 권한관리 | "권한 변경" 클릭 → 배지가 일반사용자 → 관리자로 전환 |

## i18n (규칙 G — 필수)
프로토타입(`docs/animated_demo_guide_mockup.html`)의 한글 텍스트를 그대로 옮겨선 안 된다. 모달의 모든 노출 텍스트(버튼 라벨, step 제목/설명, 진행 라벨 등)는 **반드시 i18n 키로** 처리한다.

- 위치: `frontend/src/locales/ko.json` / `en.json`
- 키 그룹: 기존 `guide.*`와 충돌하지 않도록 **하위 네임스페이스 `guide.tour.*`** 사용 권장
  - 예: `guide.tour.button`("전체 가이드"), `guide.tour.next`, `guide.tour.prev`, `guide.tour.skip`, `guide.tour.start`("시작하기"), `guide.tour.pause`("일시정지"), `guide.tour.resume`("재생"), `guide.tour.progress`("{{current}}/{{total}}")
  - step별: `guide.tour.steps.request.title` / `...description`, `approval`, `history`, `voc`, `permission`
- `ko.json` / `en.json`에 **동일 키를 반드시 동시 추가**한다.

## 인터랙션 플로우
1. **idle** — 모달 비활성 상태
2. **열림 (step 0)** — 버튼 클릭 시 "요청서 작성" 단계부터 시작, 해당 데모 애니메이션 자동 재생
3. **다음/이전** — 클릭 시 이전 단계 데모 정리 후 다음 단계 데모 시작
4. **일시정지/재생** — 일시정지 클릭 시 현재 step의 데모 애니메이션이 멈추고 버튼이 "재생"으로 전환, 재생 클릭 시 이어서 재생. step을 이동하면 일시정지 상태는 해제(기본 재생)된다.
5. **마지막 단계(권한관리)** — "다음" 버튼 텍스트가 "시작하기"로 전환, 클릭 시 모달 종료
6. **건너뛰기** — 어느 단계에서든 즉시 종료

## 모달 레이아웃 (스포트라이트 없음)
- 항상 화면 정중앙, 고정 크기 (참고: 프로토타입 기준 600px 너비)
- 배경은 단순 dim overlay (`rgba(15,17,26,0.5)`), 사이드바나 특정 UI 요소를 강조하는 cutout 없음
- 모달 내부 구성(위→아래): 진행 라벨(`1/5`) — 페이지 데모 프리뷰 프레임 — 제목/설명 — 진행 dot — 일시정지/재생 토글 — 이전/다음/건너뛰기 버튼

> 영속성 없음: 버튼 클릭 시에만 표시되는 구조이므로 "다시 보지 않기" 및 노출 여부 저장(localStorage/백엔드 필드)은 두지 않는다. 별도 모델/마이그레이션이 필요 없다.

## 접근성 / 예외 처리
- `Esc` 키로 모달 즉시 종료
- 모달 열림 시 포커스 트랩 적용 (Tab으로 모달 밖 요소 이동 불가)
- 권한상 접근 불가한 페이지가 있는 사용자에게는 해당 step을 건너뛰도록 처리
  - 현재 `Navbar.tsx`의 `ROLE_PERMISSIONS` 기준: `NONE` 역할은 홈(`/`)만 접근 가능하므로 사실상 모든 데모 step을 건너뛰어야 함. 나머지 역할(`PL`, `TE_R/P/J/O/E`, `MASTER`)은 5개 페이지 전부 접근 가능하므로 step skip이 필요 없음. → step skip 로직은 `ROLE_PERMISSIONS`와 동일한 기준으로 판단한다.
- 애니메이션이 들어간 데모이므로 `prefers-reduced-motion` 사용자 설정 감지 시 정적 미리보기로 대체하는 것을 고려

## 향후 확장 고려
- 기존 기능별 가이드는 두 갈래로 구성된다: (1) TipTap(`RichTextEditor`)으로 작성한 **HTML 텍스트 콘텐츠**, (2) `components/guideDemos/`의 **빌트인 애니메이션 데모 컴포넌트**(`GUIDE_DEMOS` 매핑). 본 "전체 가이드"는 (2)와 성격이 같으므로, **기존 `guideDemos`의 애니메이션 루프 패턴/공용 유틸을 참고·재사용**하는 것을 우선 고려한다 (중복 구현 방지). 단, 기존 데모는 요청서 작성 페이지의 세부 step(라인 공정/맵 타입 등) 단위라 콘텐츠 자체를 그대로 재사용하긴 어렵고, **구현 패턴 차원에서 공유**한다.
- 추후 실제 사용자 데이터 기반 데모(예: 로그인한 사용자의 최근 요청을 예시로 활용)로 발전 가능
