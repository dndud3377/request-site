# 의뢰서 작성 기능 가이드

> 처음 사용하는 분들을 위한 Step별 기능 상세 안내입니다.
> 의뢰서는 Step 1 → Step 2 → Step 3 → Step 4 → Step 5 순서로 작성하며, 각 Step의 내용을 모두 입력한 후 다음 Step으로 넘어가세요.

---

## Step 1 · 의뢰 상세 정보

### ⚙️ 라인/공정/조리법 선택

의뢰서 작성의 **첫 번째 필수 항목**입니다. 아래 4개 필드를 모두 선택해야 이후 항목들이 활성화됩니다.

| 필드 | 설명 |
|------|------|
| **라인 (Line)** | 해당 제품이 생산되는 라인을 드롭다운에서 선택합니다. |
| **조합법 (Process Selection)** | 라인 선택 후 해당 라인의 조합법을 자동완성 입력창에서 선택합니다. 앞 글자를 입력하면 후보 목록이 나타납니다. |
| **제품 이름 (Part ID Selection)** | 조합법 선택 후 제품명(Part ID)을 자동완성 입력창에서 선택합니다. |
| **조리법 (Process ID)** | 제품 선택 후 해당 제품의 조리법(Process ID)을 자동완성 입력창에서 선택합니다. |

> **주의**: 이 4개 항목을 모두 선택하기 전에는 의뢰 목적, Flow Chart, BB 조합 구역 입력, 고객사/협력사 등 모든 하위 항목이 비활성화 상태로 입력되지 않습니다.

---

### ⚙️ 의뢰 목적

이 의뢰가 어떤 목적으로 진행되는지 선택하는 항목입니다. **반드시 1개를 선택**해야 합니다.

- **신규**: 기존 제품 없이 완전히 새로 만드는 경우
- **차용**: 기존에 등록된 제품을 기반으로 복사(Clone)하여 만드는 경우
- **신규+차용**: 신규 생성과 차용을 모두 포함하는 경우
- **Only MAP**: MAP 정보만 등록/수정하는 경우. 선택 시 기타 목적, Flow Chart, 특이사항, BB 조합, 참조 요청서 항목이 모두 비활성화되며 Step 2만 작성하면 됩니다.

---

### ⚙️ 기타 목적

의뢰 목적과 별개로 추가적인 변경 사항이 있을 경우 선택합니다. 선택하지 않아도 됩니다.

- **Layer 추가/삭제**: J-ayer 또는 O-ayer에서 Layer를 추가하거나 삭제하는 경우. 이 항목 선택 시 **참조 요청서 선택 필드**와 **Merge 버튼**이 활성화됩니다. 참조 요청서를 선택하고 Merge 버튼을 클릭하면 해당 요청서의 J-ayer/O-ayer 데이터를 현재 의뢰서에 병합할 수 있습니다.
- **STEPSEQ 변경**: 기존 제품의 Step Sequence를 변경하는 경우
- **공법 추가/변경**: 기존 제품에 공법을 추가하거나 변경하는 경우
- **Overlay, ADI CD 추가/삭제/변경**: Overlay 또는 ADI CD 항목을 추가, 삭제 또는 변경하는 경우

---

### ⚙️ Flow Chart

공정 흐름도 정보를 입력하는 테이블입니다. 여러 흐름도 경로를 행(Row) 단위로 추가할 수 있습니다.

**각 행의 입력 필드:**

| 필드 | 설명 |
|------|------|
| **흐름도 위치 (Flow Line)** | 해당 흐름도가 속하는 라인을 드롭다운에서 선택합니다. |
| **흐름도 제품 이름 (Flow Part ID)** | 흐름도의 제품명을 자동완성 입력창에서 선택합니다. |
| **흐름도 조리법 (Flow Process ID)** | 해당 제품의 조리법을 자동완성 입력창에서 선택합니다. |
| **Step 범위** | 흐름도가 적용되는 Step의 시작값과 종료값을 각각 입력합니다. (예: Step_from ~ Step_to) |

**행 추가/삭제:**
- 하단의 **+ 행 추가** 버튼을 클릭하면 새로운 흐름도 행이 추가됩니다.
- 행이 2개 이상일 때 각 행 우측에 삭제 버튼(🗑️)이 표시됩니다.

---

### ⚙️ BB 조합 구역 입력

BB(Backbone) 참조 데이터를 어디서 가져올지 지정하는 항목입니다. Step 5의 BB 매핑 시 이 정보를 기반으로 데이터를 불러옵니다. **최소 1행 이상 입력해야 합니다.**

**각 행의 입력 필드:**

| 필드 | 설명 |
|------|------|
| **BB 위치 (BB Ref Line)** | BB 데이터를 참조할 라인을 드롭다운에서 선택합니다. |
| **BB 제품 이름 (BB Ref Part ID)** | BB 참조 제품명을 자동완성 입력창에서 선택합니다. |
| **BB 조리법 (BB Ref Process ID)** | BB 참조 제품의 조리법을 자동완성 입력창에서 선택합니다. |

**행 추가/삭제:**
- **+ 추가** 버튼으로 행을 추가하고, 행이 2개 이상일 때 삭제 버튼이 나타납니다.

---

### ⚙️ 고객사/협력사

고객사 또는 협력사 관련 정보를 입력하는 항목입니다.

| 필드 | 설명 |
|------|------|
| **고객/업체명 (Customer Name)** | 고객사 또는 협력사의 이름을 텍스트로 입력합니다. |
| **요구 사항 (Customer Requirement)** | 고객 또는 협력사의 요구 사항을 텍스트로 입력합니다. |
| **실제 생산 진행 날짜 (Production Date)** | 달력에서 실제 생산이 예정된 날짜를 선택합니다. |

---

## Step 2 · MAP 정보

### ⚙️ MAP 유형 설정

이 MAP이 어떤 방식으로 만들어지는지 선택합니다. **반드시 1개를 선택**해야 하며, 유형 변경 시 기존에 입력한 MAP 정보가 모두 초기화되므로 주의하세요.

- **NEW**: 기존 제품 없이 완전히 새로 만드는 MAP
- **CLONE**: 기존에 등록된 제품의 MAP을 복사하여 만드는 경우. 선택 시 **원본 위치/원본 제품** 필드가 활성화됩니다.
- **EXISTING**: 이미 존재하는 MAP을 그대로 사용하는 경우. 선택 시 MAP 편차, 예외 구역, X mark 변경 등 대부분의 수정 항목이 비활성화됩니다.

---

### ⚙️ 원본 위치/원본 제품

MAP 유형이 **CLONE**인 경우에만 표시되는 항목입니다. 복사할 원본 제품의 정보를 입력합니다.

| 필드 | 설명 |
|------|------|
| **원본 위치 (Source Line)** | 원본 제품이 있는 라인을 드롭다운에서 선택합니다. |
| **원본 제품 이름 (Source Part ID)** | 복사할 원본 제품명을 자동완성 입력창에서 선택합니다. |

---

### ⚙️ MAP 편차 변경

기존 MAP과 비교하여 X/Y 좌표 편차가 변경되었는지 여부를 설정하는 항목입니다. **Only C가문(Only Prodc) 설정 여부에 따라 입력 방식이 달라집니다.**

**Only C가문 = No (일반 모드):**

| 필드 | 설명 |
|------|------|
| **편차 변경 여부** | '변경 없음' 또는 '변경 있음' 중 선택합니다. '변경 있음' 선택 시 아래 필드가 활성화됩니다. |
| **X값** | X축 편차 수치를 입력합니다. (예: 0.5) |
| **Y값** | Y축 편차 수치를 입력합니다. (예: -0.3) |
| **변경 사유** | 편차가 변경된 이유를 텍스트로 입력합니다. |

**Only C가문 = Yes (C가문 모드):**
- 편차 변경은 **변경 있음으로 자동 고정**됩니다.
- **북쪽(Top)**과 **남쪽(Bottom)** 각각 X값, Y값, 변경 사유를 입력해야 합니다.
- **검증 규칙**: 북쪽과 남쪽의 X값은 절대값이 같고 부호가 반대여야 합니다. (예: 북쪽 +0.5 ↔ 남쪽 -0.5) / Y값은 북쪽과 남쪽이 동일해야 합니다.

---

### ⚙️ 예외 구역 변경

MAP의 예외 처리 구역 범위가 변경되었는지 여부를 설정합니다.

| 필드 | 설명 |
|------|------|
| **예외 구역 변경 여부** | '변경 없음' 또는 '변경 있음' 중 선택합니다. |
| **예외 구역 값 (mm)** | '변경 있음' 선택 시 활성화됩니다. 변경할 예외 구역 범위를 mm 단위로 입력합니다. |

---

### ⚙️ Only C가문 설정

제품이 C가문(Only Prodc) 방식으로 처리되는지 여부를 설정합니다. 이 설정은 MAP 편차, REV 관리 등 여러 항목의 입력 방식에 영향을 미칩니다.

| 선택값 | 설명 |
|--------|------|
| **No** | 일반 MAP 처리 방식. REV 정보 및 C가문 상세 위치 입력 항목이 비활성화됩니다. |
| **Yes** | C가문 전용 처리 방식. 아래의 추가 필드들이 활성화됩니다. |

**Yes 선택 시 추가 입력 항목:**

| 필드 | 설명 |
|------|------|
| **제품 해당 위치 (Prodc Apply Region)** | '북쪽', '중간', '남쪽' 중 해당 위치를 선택합니다. |
| **북쪽 영역 (Top)** | 북쪽에 해당하는 위치(라인), 조합법, 제품 이름을 입력합니다. **필수 항목**입니다. |
| **중간 영역 (Middle)** | '사용' 또는 '미사용'을 선택합니다. '사용' 선택 시 위치, 조합법, 제품 이름을 입력합니다. **선택 항목**입니다. |
| **남쪽 영역 (Bottom)** | 남쪽에 해당하는 위치(라인), 조합법, 제품 이름을 입력합니다. **필수 항목**입니다. |

---

### ⚙️ REV 관리

Only C가문 = Yes인 경우에만 표시되는 항목입니다. Layer별 GDS 버전 이력을 관리합니다.

| 선택값 | 설명 |
|--------|------|
| **NO** | REV 정보를 관리하지 않습니다. 선택 시 기존에 입력된 REV 정보가 초기화됩니다. |
| **YES** | REV 정보를 입력합니다. 아래 항목들이 활성화됩니다. |

**YES 선택 시 입력 방법:**
1. **Layer 선택**: 활성화된 J-ayer 행에 등록된 Layer ID 버튼 중 하나를 클릭합니다.
2. **GDS version**: 해당 Layer의 GDS 버전을 텍스트로 입력합니다. (예: v1.2)
3. **추가 버튼** 클릭 시 아래 테이블에 항목이 추가됩니다.

**추가된 항목 테이블:**
- Layer / GDS version / 삭제 버튼이 표시됩니다. 삭제 버튼(🗑️)으로 개별 항목을 제거할 수 있습니다.

---

### ⚙️ X mark 변경

MAP 상의 X 표시(불량 마크) 변경 사항을 설정합니다.

| 선택값 | 설명 |
|--------|------|
| **없음** | X mark 변경 없음 |
| **추가** | 새로운 X mark를 추가합니다. 이미지 업로드 필드가 활성화됩니다. |
| **수정** | 기존 X mark를 수정합니다. 이미지 업로드 필드가 활성화됩니다. |
| **삭제** | 기존 X mark를 삭제합니다. '특정 제품 삭제 필요' 안내 메시지가 표시됩니다. |

**이미지 업로드 방법 (추가/수정 선택 시):**
- Only C가문 = No: 단일 이미지 1개 업로드
- Only C가문 = Yes: 북쪽(Top)과 남쪽(Bottom) 이미지 각각 업로드 (필수)
- **Ctrl+V**로 클립보드 이미지를 바로 붙여넣기 할 수 있습니다.

---

### ⚙️ MAP 옵션

MAP에 적용할 추가 옵션들을 설정합니다. 각 버튼을 클릭하여 **적용/미적용** 상태를 전환합니다. 여러 개를 동시에 선택할 수 있습니다.

- **Photo Backside**: 내용
- **EDS Backside**: 내용
- **Inter**: 내용
- **TSV**: 내용
- **RF**: 내용
- **Full Chip**: 내용
- **Split**: 내용
- **ST**: 내용
- **ECC**: 내용
- **Label Side Shot**: 내용
- **HPkg Label Height**: 내용

## Step 3 · J-ayer 정보

### ⚙️ J-ayer 테이블

J-ayer(Job Layer) 데이터를 관리하는 핵심 테이블입니다. 각 행이 하나의 Layer 항목을 나타냅니다.

**테이블 컬럼 설명:**

| 컬럼 | 설명 |
|------|------|
| **No** | 자동 부여되는 행 순번입니다. |
| **체크박스** | 행을 선택하거나 전체 선택/해제합니다. 선택된 행은 일괄 비활성화에 사용됩니다. |
| **Update 날짜** | 데이터가 마지막으로 업데이트된 날짜입니다. (읽기 전용) |
| **공법 (Process ID)** | 해당 Layer의 공법 ID입니다. |
| **SP (Step Seq)** | Step Sequence 번호입니다. |
| **SD (Step Desc)** | Step 설명(Description)입니다. |
| **Layer (Layer ID)** | Layer 식별자입니다. |
| **PP** | PP 정보입니다. 값에 'plel'이 포함되면 행 배경이 옅은 노란색으로 표시됩니다. |
| **ST** | 해당 Layer의 상태를 설정합니다. 드롭다운에서 선택하며, 값에 따라 색상이 다르게 표시됩니다. |
| **요청 기준 (New or Copy)** | 해당 Layer의 처리 방식을 선택합니다. 값에 따라 행 배경색이 다르게 표시됩니다. |
| **제품 이름 (Product Name)** | 제품명입니다. |
| **STEP** | 해당 Layer의 Step 범위입니다. |
| **ID** | 항목 ID를 자동완성 입력창에서 선택합니다. |

**ST 선택값 및 색상:**

| 값 | 색상 | 설명 |
|----|------|------|
| **O** | 초록색 | 내용 |
| **O (D)** | 파란색 | 내용 |
| **O (혼용)** | 주황색 | 내용 |
| **X** | 빨간색 | 내용 |
| *(빈값)* | 없음 | 미설정 |

**요청 기준 선택값 및 행 색상:**

| 값 | 행 색상 | 설명 |
|----|---------|------|
| **신규** | 흰색 | 내용 |
| **차용** | 파란색 배경 | 내용 |
| **기등록** | 회색 배경 | 이미 등록된 항목으로 읽기 전용 처리됩니다. |
| **layer삭제** | 노란색 배경 | 내용 |
| *(빈값)* | 없음 | 미설정 |

**일괄 설정 기능 (테이블 상단 툴바):**

| 버튼 | 설명 |
|------|------|
| **모두 O / 모두 X / 초기화** | 모든 행의 ST 값을 일괄 설정하거나 초기화합니다. |
| **모두 신규 / 모두 차용 / 초기화** | 모든 행의 요청 기준 값을 일괄 설정하거나 초기화합니다. |
| **STEP 정렬** | SP(Step Seq) 기준으로 행을 오름차순 정렬합니다. 다시 클릭하면 해제됩니다. |
| **필터 버튼** | 저장된 필터를 클릭하면 해당 조건의 행들이 비활성화됩니다. 다시 클릭하면 복원됩니다. |
| **+ 필터** | 새로운 필터를 생성합니다. (→ 필터 관리 기능 참조) |
| **비활성화** | 체크박스로 선택한 행들을 비활성화합니다. 비활성화된 행은 의뢰 대상에서 제외됩니다. |
| **+ 행 추가** | 새로운 빈 행을 테이블 하단에 추가합니다. |

**Excel 붙여넣기:**
- 셀을 클릭하여 선택한 후 **Ctrl+V**를 누르면 클립보드의 표 데이터를 해당 위치부터 붙여넣기 할 수 있습니다.

---

### ⚙️ 필터 관리

J-ayer 테이블에서 특정 조건의 행들을 빠르게 비활성화하기 위한 필터를 저장하고 관리합니다.

**필터 생성 방법:**
1. J-ayer 테이블 툴바의 **+ 필터** 버튼을 클릭하면 필터 관리 모달이 열립니다.
2. **필터명**을 입력합니다. (예: "특정 Layer 제외")
3. 필터 기준 키워드를 아래 항목에 추가합니다.

**필터 기준 (SP / SD / PP 키워드):**

| 기준 | 설명 |
|------|------|
| **SP 키워드** | SP(Step Seq) 컬럼 값에 해당 키워드가 포함된 행을 필터링합니다. |
| **SD 키워드** | SD(Step Desc) 컬럼 값에 해당 키워드가 포함된 행을 필터링합니다. |
| **PP 키워드** | PP 컬럼 값에 해당 키워드가 포함된 행을 필터링합니다. |

- 키워드는 각 항목에 **여러 개** 추가할 수 있습니다.
- 저장된 필터는 툴바에 버튼으로 표시되며, 버튼을 클릭하면 해당 조건의 행들이 즉시 비활성화됩니다.
- 필터 버튼을 다시 클릭하면 비활성화가 해제됩니다.
- 필터 관리 모달에서 저장된 필터를 삭제할 수 있습니다.

---

## Step 4 · O-ayer 정보

### ⚙️ O-ayer 테이블

O-ayer(Overlay Layer) 데이터를 관리하는 테이블입니다. J-ayer 테이블과 동일한 구조로 운영됩니다.

**탭 구성:**
- **OVL Layer 목록**: O-ayer 테이블이 표시됩니다.
- **OVL 정보**: Partial Shot 및 TBV/TLV 설정 항목이 표시됩니다.

**테이블 컬럼 및 기능:**
J-ayer 테이블과 동일합니다. (컬럼: No, 체크박스, Update 날짜, 공법, SP, SD, Layer, PP, ST, 요청 기준, 제품 이름, STEP)

- ST 선택값(O / O(D) / O(혼용) / X)과 색상 코딩이 동일하게 적용됩니다.
- 요청 기준(신규 / 차용 / 기등록 / layer삭제)과 행 색상 코딩이 동일하게 적용됩니다.
- 일괄 설정, STEP 정렬, 필터, 행 추가, 비활성화 기능 모두 동일합니다.
- Excel 붙여넣기(**Ctrl+V**)도 동일하게 지원됩니다.

---

### ⚙️ Partial Shot

OVL 정보 탭에서 설정하는 항목입니다. **필수 선택**입니다.

- **O**: 내용
- **X**: 내용

---

### ⚙️ TBV/TLV 설정

O-ayer 테이블에 SD 값이 TBV 또는 TLV를 포함하는 행이 존재할 때만 OVL 정보 탭에 표시됩니다.

**입력 방법:**
1. **두께 (Thickness)**: 전체 항목에 공통으로 적용될 두께 수치를 입력합니다.
2. **SD 선택**: 테이블에서 확인된 TBV/TLV SD 값들이 버튼으로 표시됩니다. 추가할 SD 버튼을 클릭하여 선택합니다.
3. **비고 (Note)**: 해당 항목에 대한 메모를 입력합니다.
4. **추가 버튼** 클릭 시 선택한 SD와 비고가 아래 테이블에 추가됩니다.

> **주의**: 두께나 비고를 입력하지 않고 추가하면 경고 모달이 나타납니다. 계속 진행할지 확인 후 추가하세요.

**추가된 항목 테이블:**
- SD / 비고 / 삭제 버튼이 표시됩니다. 삭제 버튼(🗑️)으로 개별 항목을 제거할 수 있습니다.

---

## Step 5 · 뼈찜(BB) 정보

### ⚙️ BB 자동 채우기

J-ayer 행과 BB 데이터를 Layer 범위를 기준으로 **자동으로 매핑**하는 기능입니다. 수동으로 하나씩 매핑하기 전에 이 기능을 먼저 사용하면 효율적입니다.

**사용 방법:**
1. **📋 Backbone 자동 채움** 버튼을 클릭합니다.
2. Layer 범위 설정 패널이 열립니다.
3. 각 범위를 아래와 같이 설정합니다.

| 필드 | 설명 |
|------|------|
| **시작 Layer** | 자동 채우기를 적용할 J-ayer의 시작 Layer ID를 드롭다운에서 선택합니다. |
| **종료 Layer** | 자동 채우기를 적용할 J-ayer의 종료 Layer ID를 드롭다운에서 선택합니다. |
| **제품 ID** | 해당 범위에 사용할 BB 참조 제품(Step 1에서 입력한 BB 조합 구역 목록)을 선택합니다. |

4. **+ 범위 추가** 버튼으로 여러 범위를 동시에 설정할 수 있습니다.
5. 설정 완료 후 **✔ 적용** 버튼을 클릭하면 범위 내 J-ayer 행들에 BB 데이터가 자동으로 채워집니다.

---

### ⚙️ BB 매핑

J-ayer 각 행에 BB 데이터를 **직접 선택하여 매핑**하는 기능입니다. 자동 채우기로 처리되지 않은 행이나 수동으로 조정이 필요한 경우 사용합니다.

**화면 구성 (좌우 분할):**

**왼쪽 패널 - J-ayer 목록:**
- 활성화된 J-ayer 행들이 표시됩니다.
- 각 행에는 공법 / STEPSEQ / Step 설명 / Layer / 현재 BB 매핑 상태가 표시됩니다.
- 이미 BB가 매핑된 행은 목록에서 제외됩니다.
- 행을 클릭하면 선택 상태가 됩니다.

**오른쪽 패널 - BB 데이터 목록:**
- Step 1에서 입력한 BB 조합 구역 탭별로 데이터가 구분됩니다.
- 상단 검색 필드에서 **공법 / 제품명 / SEQ / Step 설명 / Layer** 키워드로 데이터를 필터링할 수 있습니다.

**매핑 방법:**
1. 왼쪽에서 매핑할 J-ayer 행을 클릭합니다.
2. 오른쪽에서 연결할 BB 데이터 행을 클릭합니다. (파란색으로 하이라이트됩니다)
3. 여러 쌍을 계속 선택할 수 있습니다.
4. **✔ 적용 (N건)** 버튼을 클릭하면 선택한 매핑이 일괄 적용됩니다.
5. 매핑된 행은 왼쪽 목록에서 사라지고 BB 적용 테이블에 추가됩니다.
6. 매핑을 취소하려면 J-ayer 행의 X 버튼을 클릭합니다.

---

### ⚙️ BB 적용 테이블

BB 매핑이 완료된 결과를 확인하고 수정하는 테이블입니다.

**테이블 컬럼 설명:**

| 컬럼 | 설명 |
|------|------|
| **체크박스** | 행을 선택합니다. 선택된 행은 일괄 원복 삭제에 사용됩니다. |
| **No** | 자동 순번입니다. |
| **공법 (Process ID)** | J-ayer의 공법 ID입니다. |
| **SEQ (SS)** | Step Seq 값입니다. 헤더를 클릭하면 오름차순으로 정렬됩니다. |
| **SD** | Step 설명(Description)입니다. |
| **BB 공법 (BB Process ID)** | 매핑된 BB의 공법 ID입니다. |
| **BB 제품명 (BB Part ID)** | 매핑된 BB의 제품명입니다. |
| **BB Layer** | 매핑된 BB의 Layer ID입니다. |
| **BB STEPSEQ** | 매핑된 BB의 Step Seq 값입니다. |
| **비고 (Remark)** | 해당 항목에 대한 메모를 입력할 수 있습니다. |

**주요 기능:**

| 기능 | 설명 |
|------|------|
| **셀 직접 수정** | 모든 셀을 클릭하여 직접 수정할 수 있습니다. |
| **+ 행 추가** | 새로운 빈 행을 추가합니다. 자동 채우기나 매핑으로 처리되지 않은 항목을 수동으로 입력할 때 사용합니다. |
| **선택 원복 (N개)** | 체크박스로 선택한 행들을 삭제하고 BB 매핑을 원래 상태로 되돌립니다. |
| **🗑️ 초기화** | 테이블의 모든 BB 데이터를 삭제하고 처음부터 다시 시작합니다. |
